# LinkedIn Automation SaaS — Complete Modular Plan

## Project: LinkedBoost (Working Title)
**Stack**: Next.js 16 (App Router) + React 19 + Tailwind CSS v4 + MongoDB Atlas + NextAuth.js + Chrome Extension (Manifest V3)
**Deployment**: Self-hosted VPS (Docker)
**Monetization**: Completely free forever (BYOK — users bring their own AI API keys)
**Goal**: Best-in-class LinkedIn automation tool that goes viral

## MODULE 2: Database Schema & MongoDB Setup

**Purpose**: Design and implement all MongoDB collections, indexes, and data models using Mongoose.

**Collections to create**:

1. **User** — Core user document
   - `_id`, `name`, `email`, `image`, `hashedPassword`
   - `linkedinProfile`: { name, headline, url, profilePicUrl }
   - `aiApiKeys`: { provider: enum[gemini|openai|anthropic|groq], encryptedKey: string, isValid: boolean }[] — AES-256-GCM encrypted
   - `settings`: { timezone, language, notificationPrefs, dailyLimits: { applies, posts, scrapes } }
   - `subscription`: { plan: 'free', startDate } — future-proofing even if free
   - `stats`: { totalApplied, totalPosts, totalScraped, successRate }
   - `createdAt`, `updatedAt`
   - **Indexes**: email (unique), createdAt

2. **Resume** — User's resume data
   - `userId` (ref User), `name`, `isDefault: boolean`
   - `contactInfo`: { phone, email, location, linkedin, github, portfolio }
   - `summary`: string
   - `experience`: [{ company, title, startDate, endDate, current, description, highlights[] }]
   - `education`: [{ school, degree, field, startDate, endDate, gpa }]
   - `skills`: string[]
   - `certifications`: [{ name, issuer, date }]
   - `projects`: [{ name, description, url, tech[] }]
   - `rawText`: string (original parsed resume text for AI context)
   - `pdfUrl`: string (stored PDF link)
   - **Indexes**: userId, isDefault

3. **JobSearch** — Saved search configurations
   - `userId`, `name`, `keywords`, `location`, `remote: boolean`
   - `experienceLevel`: enum[], `datePosted`: enum
   - `easyApplyOnly: boolean`, `salary`: { min, max }
   - `excludeCompanies`: string[], `excludeKeywords`: string[]
   - `isActive: boolean`, `schedule`: cron expression (for recurring searches)
   - **Indexes**: userId, isActive

4. **JobApplication** — Individual job application records
   - `userId`, `jobSearchId` (ref JobSearch)
   - `jobTitle`, `company`, `location`, `jobUrl`, `jobDescription`
   - `status`: enum[found|tailoring|applying|applied|failed|skipped|interview|rejected|offered]
   - `tailoredResume`: { summary, skills[], highlights[] } — AI-generated modifications
   - `formAnswers`: [{ question, answer, fieldType }]
   - `appliedAt`, `notes`
   - `matchScore`: number (AI-calculated job-resume match %)
   - **Indexes**: userId + status (compound), appliedAt, company

5. **HeroProfile** — "Become a Hero" configuration
   - `userId`, `niche`: string, `targetAudience`: string
   - `contentPillars`: string[] (3-5 themes to post about)
   - `postingSchedule`: { days: number[], timesPerWeek: number, preferredTimes: string[] }
   - `groups`: [{ groupId, name, url, joined: boolean, joinedAt }]
   - `hashtags`: string[] (tracked hashtags)
   - `voiceTone`: enum[professional|casual|inspirational|educational|humorous]
   - `contentQueue`: [{ content, scheduledFor, status }]
   - **Indexes**: userId

6. **Post** — LinkedIn post records
   - `userId`, `heroProfileId`
   - `content`: string, `type`: enum[text|image|carousel|poll|video|article]
   - `hashtags`: string[], `targetGroups`: string[]
   - `status`: enum[draft|scheduled|posting|posted|failed]
   - `scheduledFor`: Date, `postedAt`: Date
   - `engagement`: { views, likes, comments, shares, impressions }
   - `linkedinPostUrl`: string
   - **Indexes**: userId + status, scheduledFor

7. **ScrapedData** — Scraped LinkedIn data
   - `userId`, `scraperConfigId`
   - `type`: enum[post|profile|company|job]
   - `data`: Mixed (flexible schema for different data types)
   - `source`: { url, scrapedAt }
   - `tags`: string[]
   - `actions`: [{ type: enum[commented|reached_out|saved|dismissed], at: Date, content: string }]
   - **Indexes**: userId + type, scrapedAt, tags

8. **ScraperConfig** — Scraper search configurations
   - `userId`, `name`, `type`: enum[posts|profiles|companies]
   - `keywords`: string[], `filters`: Mixed
   - `schedule`: cron, `isActive: boolean`
   - `maxResults`: number
   - **Indexes**: userId, isActive

9. **ActivityLog** — All automation activity for audit
   - `userId`, `action`: string, `module`: enum[jobs|hero|scraper]
   - `details`: Mixed, `status`: enum[success|failure|skipped]
   - `timestamp`, `linkedinUrl`: string
   - **TTL Index**: timestamp (auto-delete after 90 days)
   - **Indexes**: userId + module, timestamp

10. **DailyUsage** — Rate limiting and daily usage tracking
    - `userId`, `date`: Date (YYYY-MM-DD)
    - `actions`: { applies: number, posts: number, scrapes: number, profileViews: number, messages: number }
    - **Indexes**: userId + date (compound unique)

**Performance**:
- All queries should use covered indexes
- Use MongoDB aggregation pipeline for analytics
- Enable MongoDB connection pooling (max 10 connections)
- Use lean() for read-only queries (skip Mongoose hydration)
- Implement cursor-based pagination, never skip-limit for large datasets

## MODULE 5: Chrome Extension Foundation

**Purpose**: Create the Chrome Extension (Manifest V3) that operates on LinkedIn pages within the user's browser.

**What to implement**:

1. **Manifest.json**:
   ```json
   {
     "manifest_version": 3,
     "name": "LinkedBoost",
     "version": "1.0.0",
     "permissions": ["activeTab", "storage", "tabs", "scripting", "alarms"],
     "host_permissions": ["https://www.linkedin.com/*"],
     "background": { "service_worker": "background.js", "type": "module" },
     "content_scripts": [{
       "matches": ["https://www.linkedin.com/*"],
       "js": ["content.js"],
       "run_at": "document_idle"
     }],
     "action": { "default_popup": "popup.html" }
   }
   ```

2. **Background Service Worker** (`extension/background/`):
   - WebSocket connection to Next.js app for receiving commands
   - Message relay between web app and content script
   - Chrome alarm-based scheduling (for periodic tasks)
   - Connection state management (reconnect with exponential backoff)
   - Command queue (buffer commands when content script not ready)

3. **Content Script** (`extension/content/`):
   - LinkedIn DOM interaction layer
   - Page detection: which LinkedIn page is the user on? (feed, jobs, profile, messaging)
   - DOM mutation observer for SPA navigation detection
   - Action executors: click, type, scroll, upload, select
   - All actions wrapped in anti-detection layer (Module 7)

4. **Extension ↔ Web App Communication**:
   - WebSocket server in Next.js API route (using Socket.IO)
   - Auth token exchange: extension gets a temporary token from web app
   - Message types: `EXECUTE_ACTION`, `REPORT_STATUS`, `HEARTBEAT`, `SYNC_CONFIG`
   - Binary protocol for efficient data transfer

5. **Extension Popup UI**:
   - Simple React popup (built with Vite for fast builds)
   - Shows: connection status, current task, pause/resume buttons
   - "Open Dashboard" link to web app
   - Login prompt if not authenticated

6. **Extension Build System**:
   - Vite config for extension build (separate from Next.js)
   - Content script bundled separately (injected into LinkedIn pages)
   - Background service worker bundled with all dependencies
   - Hot reload during development

**Anti-Detection (Critical)**:
- Content script does NOT modify LinkedIn's DOM suspiciously
- All interactions go through native event dispatching (not synthetic)
- No `document.querySelector` patterns that LinkedIn can detect
- Use `MutationObserver` instead of polling for DOM changes
- Extension fingerprint is that of a normal browser extension

## MODULE 7: Anti-Detection Engine

**Purpose**: Build the core anti-detection system that makes all LinkedIn automation appear human.

**What to implement**:

1. **Human Behavior Simulator** (`lib/anti-detection/human-simulator.ts`):
   - `randomDelay(min, max)` — Gaussian-distributed delays (not uniform random)
   - `humanType(element, text)` — Type with variable keystroke delays (50-250ms), occasional typos + corrections
   - `humanClick(element)` — Move mouse in Bezier curve path to element, then click
   - `humanScroll(distance)` — Scroll in chunks with variable speed, pause at content
   - `readingPause(contentLength)` — Simulate reading time based on content length (200-300 WPM)

2. **Action Rate Limiter** (`lib/anti-detection/rate-limiter.ts`):
   - Daily action budget per category:
     - Job applications: max 15/day (safe limit)
     - Posts: max 2/day
     - Profile views: max 30/day
     - Connection requests: max 20/day
     - Comments: max 15/day
     - Messages: max 10/day
   - Cooldown periods between same-type actions (5-15 min for applies)
   - Session duration limits (max 2-3 hours active, then break)
   - Weekend/holiday awareness (reduce activity)
   - Progressive warm-up: new accounts start with 50% limits, increase over 2 weeks

3. **Session Manager** (`lib/anti-detection/session-manager.ts`):
   - Random session start times (don't always start at same time)
   - Natural session durations (30min - 2hr)
   - Break patterns (browse feed for 5min between job applications)
   - End-of-session cooldown (gradually reduce activity speed)

4. **Activity Patterns** (`lib/anti-detection/patterns.ts`):
   - Mix automated actions with "browsing" actions (scroll feed, view profiles)
   - Randomize the order of job applications (not sequential)
   - Vary the time spent on each job posting (30s - 3min)
   - Occasionally skip jobs (human wouldn't apply to every single one)
   - Interact with feed between applies (like a post, read an article)

5. **Detection Avoidance**:
   - Monitor for LinkedIn CAPTCHA/verification prompts
   - Auto-pause if unusual page behavior detected (rate limit page, security check)
   - Alert user immediately if account warning detected
   - Track and log all suspicious responses from LinkedIn

**Configuration** (user-adjustable):
- Speed: Conservative / Balanced / Aggressive
- Working hours: Set active hours (e.g., 9am-5pm)
- Days active: Select days of week
- Max daily actions per category

## MODULE 9: Job Search & Easy Apply Automation

**Purpose**: The core feature — search for jobs on LinkedIn, find Easy Apply listings, tailor resume, fill forms, and apply automatically.

**What to implement**:

1. **Job Search Configuration UI** (`app/(dashboard)/jobs/page.tsx`):
   - Search form: keywords, location, remote/hybrid/onsite, experience level
   - Filters: date posted, salary range, company exclusions, keyword exclusions
   - Easy Apply only toggle (default ON)
   - Save search configurations for recurring use
   - Schedule searches (run daily at specific time)

2. **Job Discovery** (Extension content script):
   - Navigate to LinkedIn Jobs with search parameters
   - Scroll through results, extracting:
     - Job title, company, location, salary (if shown), posted date
     - Easy Apply badge detection
     - Job URL and unique ID
   - Pagination handling (load more results)
   - Send discovered jobs back to web app via WebSocket

3. **Job Analysis & Scoring** (Server-side):
   - Receive job listings from extension
   - AI-powered match scoring: compare job description vs user's resume
   - Filter jobs below match threshold (user-configurable, default 60%)
   - Rank jobs by match score
   - Detect duplicate applications (already applied)
   - Present scored jobs in dashboard for user review

4. **Automated Application Flow** (Extension):
   - For each approved job (auto or manual approval):
     a. Navigate to job posting
     b. Click "Easy Apply" button
     c. Detect form fields on each page:
        - Text inputs → fill from resume data
        - Dropdowns → select matching option
        - Radio buttons → select based on AI analysis
        - File upload → upload tailored resume PDF
        - Custom questions → AI generates answers
     d. Handle multi-step forms (detect "Next" button, navigate pages)
     e. Review step → verify all fields filled
     f. Submit application
     g. Record result (success/failure) with screenshot
   - Between applications: random delays (5-15 min), feed browsing

5. **Job Application Dashboard** (`app/(dashboard)/jobs/applications/page.tsx`):
   - Table/Kanban view of all applications
   - Status tracking: Found → Tailoring → Applying → Applied → Interview → etc.
   - Filter by status, company, date, match score
   - Click to view job details, tailored resume, form answers
   - Export to CSV
